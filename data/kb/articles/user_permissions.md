# KB-004: User Access and Permission Issues

## Symptoms
- User receives "Access Denied" message
- Certain features or reports are unavailable
- User cannot view or edit specific data

## Likely Causes
- Incorrect role or permission assignment
- Recent changes to access control settings
- User not assigned to correct team or group

## Resolution Steps
1. Verify the user’s assigned role and permissions
2. Compare with a working user in the same role
3. Update permissions if necessary
4. Ask user to log out and log back in
5. Confirm access after permission changes

## Escalate If
- Permissions appear correct but issue persists
- Access issues affect multiple users
- Changes do not apply after system refresh
